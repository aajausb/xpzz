## 上午 (08:30-11:46) — 私钥签名问题讨论

### OKX OnchainOS 签名机制确认
- 跑步哥发了OKX官方教程推文 (okxchinese/status/2028806839773839683)
- 教程说"5步接入"但没提到签名问题
- 实测：onchainos swap swap 返回 tx.data + signatureData=[''] (空)
- 实测：直接把 tx.data 当 signed-tx 广播 → orderId 返回成功但链上拒绝 "Transaction did not pass signature verification"
- **结论：OKX API Key ≠ 私钥。API Key 是调用 OKX 服务的通行证，私钥是链上签名用的，两者完全独立**
