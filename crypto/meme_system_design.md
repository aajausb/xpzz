### 🧠 聪明钱 = 系统核心

聪明钱不是辅助，是**最核心的信号源**：
- **主动信号**：聪明钱买了一个币 → 直接触发分析流程（最高优先级）
- **被动验证**：新币上线 → 查前20买家有无聪明钱 → 有则评分+20
- **卖出参考**：聪明钱开始卖持仓币 → 触发跟卖逻辑

聪明钱列表是**私有资产**，自建不公开：
- 从链上扫描：7天内 pump.fun 盈利>$5k、胜率>60%、交易>20笔
- 每3天自动重扫更新
- 不用GMGN/Birdeye等公开列表（已被跟烂）

## 三、过滤层（去垃圾）

99%的新币是垃圾，必须严格过滤：

### 3.1 安全检查（一票否决）
```
❌ 不买：
  - Dev持仓 > 5%（可以随时砸盘）
  - 合约未放弃ownership（可以mint新币）
  - LP未锁定（可以随时撤池子 = rug pull）
  - 蜜罐检测阳性（能买不能卖）
  - 同一Dev过去创建的币全归零（惯犯）
```

### 3.2 Dev钱包分析
```
查Dev历史：
  - 过去创建了多少币？
  - 有没有成功案例？（至少1个涨过5x）
  - 创建后多久开始卖？（<1小时 = 高危）
  - 是否用多个钱包分散持仓？（隐藏庄家行为）
```

### 3.3 早期买家分析
```
前20个买家是谁？
  ├─ 有庄家钱包参与 → 加分
  ├─ 买家之间有关联（同源资金）→ 减分（假热度）
  ├─ 买入金额分布均匀 → 加分（真社区）
  └─ 单一大户占比>20% → 减分（容易砸）
```

### 3.4 流动性检查
```
  - 初始流动性 > 5 SOL（太小容易被操纵）
  - bonding curve 进度（>10% = 有人在买）
  - 买卖比 > 2:1（买的人多于卖的）
```

## 四、决策层（评分模型）

### 4.1 评分卡（满分100）
```
安全分（必须≥30，否则一票否决）
  ├─ 合约安全     15分  (ownership放弃+LP锁定+非蜜罐)
  ├─ Dev信誉      15分  (历史成功率+持仓比例)

热度分（衡量市场关注度）
  ├─ 庄家参与     20分  (私有列表里的钱包买了)
  ├─ 买家数量     10分  (前10分钟>50个独立买家)
  ├─ 买入速度     10分  (每分钟新买家增速)

质量分（衡量币本身质量）
  ├─ 叙事/名字    10分  (AI/动物/热梗/政治 = 高分)
  ├─ 流动性       10分  (初始池子大小)
  └─ bonding进度  10分  (进度快 = 有热度)

总分 ≥ 70 → 买入信号
总分 ≥ 85 → 加大仓位
```

### 4.2 触发条件（按优先级排序）
```
条件A（聪明钱主动买入 — 最强信号）:
  私有聪明钱买入 + 安全通过 + 评分≥60 → 买入$20
  多个聪明钱买同一币 → 加仓到$30

条件B（新币+聪明钱验证）:
  新币上线<15分钟 + 前20买家有聪明钱 + 评分≥70 → 买入$20

条件C（新币纯评分）:
  新币上线<15分钟 + 无聪明钱但评分≥80 → 买入$15（低一档）

条件D（毕业币）:
  pump.fun毕业到Raydium + 评分≥65 → 买入$25
```

## 五、执行层（买卖操作）

### 5.1 买入
```
信号触发
  → 最终安全检查（双重确认）
  → Jupiter swap API 获取路由
  → 构建交易（priority fee + skipPreflight）
  → Helius RPC 发送
  → 确认成功 → 记录持仓
  → 失败 → 重试1次 → 放弃
```

### 5.2 持仓监控（每5秒）
```
每个持仓：
  → 查当前价格（Jupiter price API / Raydium）
  → 计算盈亏比
  → 触发卖出条件检查
```

### 5.3 卖出规则
```
止损：
  -50% → 全卖（无条件，不抱幻想）

阶梯止盈：
  2x  → 卖40%（回本）
  5x  → 卖30%（锁利润）
  10x → 卖20%（大肉）
  剩10% → 让它飞（moonbag）

紧急卖出：
  - 检测到Dev大量卖出 → 全卖
  - 检测到LP被撤 → 全卖
  - 价格5分钟内跌>30% → 全卖
```

## 六、管理层

### 6.1 资金管理
```
总资金：~1.1 SOL（≈$180）
  ├─ 单笔买入：$20（≈0.12 SOL）
  ├─ 最大同时持仓：5个
  ├─ 每日最大亏损：$50 → 暂停1小时
  └─ 总亏损熔断：$100 → 停机通知跑步哥
```

### 6.2 通知
```
Telegram 通知跑步哥：
  ├─ 🟢 买入：币名、金额、评分、原因
  ├─ 🔴 卖出：币名、盈亏、持有时间
  ├─ ⚠️ 告警：日亏损>$50、熔断、异常
  └─ 📊 日报：当日交易次数、总盈亏、最佳/最差
```

### 6.3 数据记录
```
每笔交易记录：
  ├─ 买入时间、价格、数量、评分
  ├─ 卖出时间、价格、盈亏
  ├─ 信号来源（新币/庄家/毕业）
  └─ 用于回溯优化策略
```
