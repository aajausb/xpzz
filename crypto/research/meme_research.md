# 土狗研究笔记

## 研究时间: 2026-03-14

## 1. 关键发现

### Reddit 用户真实经验（重要）
一个尝试做 Solana meme bot 的人的总结：
> "大多数策略保质期很短。这周有效的下周就失效了。因为顶层自动化玩家不断进化。越复杂的条件越容易失效，反而越简单的条件越有效，但需要极快的执行速度。"

**启示**：不要做复杂策略，做简单但快的。

### 2026 Meme 交易策略框架（tradingsignalzone.com）
- **两阶段市场周期**：吸筹期（低波动、持仓人数稳定增长）→ 拉盘期（爆发式上涨）
- **仓位管理**：总 meme 仓位不超过总资产 10%，单笔不超过 2%
- **筛选条件**：流动性 > $500k，市值 < $50M，持仓分布不能太集中
- **止盈**：2x 卖一部分，5x 再卖，10x 再卖
- **止损**：-30% 硬止损，不抱幻想

### 实操流程（Reddit 交易者）
1. 开 Photon（交易）+ DexScreener（图表）+ RugCheck（安全）
2. 看 30min/1h 涨幅榜
3. 筛选：流动性锁定、几百个 maker、有回调但还在涨
4. 避开暴涨后暴跌 90% 的币
5. 买入后盯盘不离手

## 2. 我们的优势和劣势

### 优势
- 24 小时不间断监控
- 有 3 条链的基础设施（Solana/BSC/Base）
- 有聪明钱数据库（Solana 543 + BSC 253 + Base 84）
- 能自动执行，不受情绪影响

### 劣势
- 信号源被稀释（公开平台聪明钱列表alpha已耗尽）
- 之前 69% 归零率
- 没有情绪感知能力（推特/TG 群热度）
- $100 起步资金极小

## 3. 策略方向

### 方向 A：庄家行为追踪（链上）
- 不用公开聪明钱列表，自建私有列表
- 监控新出现的高胜率钱包（过去 7 天胜率 > 60%、利润 > $10k）
- 跟踪这些钱包的买入行为
- 优点：纯数据驱动，可自动化
- 缺点：滞后，庄家可能用新钱包

### 方向 B：叙事驱动（情绪面）
- 监控推特/TG 热词爆发
- 检测到某个币名突然被大量提及 → 检查链上数据 → 买入
- 优点：能抓到情绪起来的第一波
- 缺点：需要推特 API、可能被假信息骗

### 方向 C：新币筛选（pump.fun 风格）
- 监控 pump.fun / Raydium 新上线的币
- 实时筛选：bonding curve 进度、holder 数量增速、DEV 持仓
- 买入满足条件的币，快进快出
- 优点：最早期，利润空间大
- 缺点：99% 是垃圾，需要极高的过滤精度

### 方向 D：混合策略（推荐）
- 用 C 监控新币
- 用 A 过滤（有庄家钱包参与的优先）
- 用 B 确认（有社交热度的优先）
- 三重验证才下单

## 7. 选币评分模型 v1（2026-03-14 14:00 设计）

### 设计原则
- 简单 > 复杂（Reddit经验：越复杂越容易失效）
- 速度 > 精度（毫秒级响应，宁可误杀不可漏防）
- 数据驱动，不凭感觉
- $20/笔小仓位，允许高误判率，只要偶尔10x就赚

### 评分维度（总分100分，≥60分买入）

#### A. 安全基础（30分，硬性淘汰项）
| 检查项 | 分值 | 标准 | 工具 |
|--------|------|------|------|
| 非貔貅盘 | 10 | 能正常卖出 | RugCheck / honeypot.is |
| DEV持仓≤5% | 10 | 单一地址不超过5% | getTokenLargestAccounts |
| 流动性≥$10k | 5 | 至少能出得去 | DexScreener |
| 非已知scam模板 | 5 | 合约不在黑名单 | RugCheck |

**任何一项不过=直接淘汰，不看其他分数**

#### B. 链上信号（40分，核心）
| 指标 | 分值 | 标准 | 数据源 |
|------|------|------|--------|
| 庄家/聪明钱参与 | 15 | 我们库里的钱包买了 | smart_wallets.json |
| Holder增长率 | 10 | 1h内新增≥50个holder | Helius getTokenAccounts |
| 买入/卖出比 | 10 | buy:sell ≥ 3:1（前30min） | Helius parsedTransactions |
| 大单买入 | 5 | 单笔≥1 SOL的买入≥5笔 | Helius parsedTransactions |

#### C. 市场结构（15分）
| 指标 | 分值 | 标准 | 数据源 |
|------|------|------|--------|
| bonding curve进度 | 5 | 20%-70%（太低没人买，太高快毕业） | pump.fun API |
| 市值合理区间 | 5 | $50k-$5M（太小流动性差，太大倍数低） | DexScreener |
| 大盘环境 | 5 | SOL 24h涨幅≥0%（不逆势） | CoinGecko |

#### D. 情绪加分（15分，锦上添花）
| 指标 | 分值 | 标准 | 数据源 |
|------|------|------|--------|
| 推特提及 | 10 | 搜到≥3条非bot推文 | agent-reach Twitter搜索 |
| 叙事热点 | 5 | 符合当前热门叙事（AI/政治/动物） | 人工维护热词表 |

### 动态权重
- **庄家信号权重翻倍**：如果聪明钱库里≥2个钱包买了同一个币，B部分庄家分数×2（最高30分）
- **速度衰减**：币创建超过4小时，总分×0.8；超过24小时，×0.5
- **大盘惩罚**：SOL 24h跌幅>5%，总分×0.7（熊市别冲）

### 实际执行流程
```
新币出现 → 安全检查(A) → 不过直接跳过
         ↓ 过了
  链上信号(B) + 市场结构(C) 并行计算（<2秒）
         ↓
  总分≥45 → 推特快查(D)（<5秒）
         ↓
  总分≥60 → 下单$20
  总分45-60 → 加入观察，5分钟后重新评分
  总分<45 → 跳过
```

### 止盈止损（买入后）
| 条件 | 动作 |
|------|------|
| +100%（2x）| 卖40%，保本 |
| +400%（5x）| 再卖30%，锁利 |
| +900%（10x）| 再卖20%，剩10%放飞 |
| -50% | 全清，不补仓 |
| 庄家钱包开始卖 | 立即跟卖全部 |
| 创建>24h且没涨 | 全清 |

### 模型验证方法
- 用过去30天已涨10x的币（WAR/SMITH等）回测：这些币在早期能打多少分？
- 用过去30天归零的币回测：这些币能被筛掉吗？
- 目标：10x币评分≥60，归零币评分<45

- 推特搜索有延迟（5-10秒），可能错过最快的窗口
- 聪明钱库还在建设中（scanner v5运行中，当前≤5个验证钱包）

## 8. 庄家吸货链上特征分析（2026-03-14 15:10 更新）

### 数据来源
- Pine Analytics pump.fun 操纵报告（2025-04）
- Medium @jump_bit 内幕指南（2025-12）
- Cryptovate sniper wallet 调查
- Scanner v5 实际扫描结果

### 核心模式一：Sniper Wallet（狙击钱包）
**规模**：pump.fun 约1.75%的发币有sniper操纵（月均>15,000个），87%成功率，利润>15,000 SOL/月
**特征**：
1. **资金来源追溯**：sniper钱包由token部署者（DEV）通过proxy/burner钱包资助
2. **时间特征**：主要活跃在美国时段（UTC 14:00-04:00）
3. **行为模式**：
   - 在token创建的同一区块或下一区块买入（block+0 或 block+1）
   - 买入后持有极短（分钟级），等散户FOMO进场后dump
   - 单笔利润200x-800x
4. **反检测手段**：
   - 用proxy钱包中转资金，切断DEV↔sniper直接链接
   - 使用burner钱包（用完即弃）
   - 分散资金跨多个钱包

### 核心模式二：Bundler（捆绑买入）
**定义**：一个操作者控制30-50个钱包，在同一笔交易内同时买入，伪造"多人同时看好"
**特征**：
1. **持仓分布伪装**：单地址40%是红旗，但40个地址各1%看起来很健康
2. **检测方法**：
   - 查所有early buyer的资金来源（SOL从哪来？）
   - 如果多个buyer的SOL来自同一源头 → bundle
   - 交易时间戳完全相同（同一slot）→ bundle
   - 后续卖出时间高度同步 → bundle
3. **Scanner v5验证**：Bird币8个地址各持仓6.22%（等额分配 = 典型bundle/预售）

### 核心模式三：庄家吸货（我们要跟的目标）
**与scam区别**：庄家吸货是有计划的低位建仓，不是立即dump
**特征**：
1. **时间跨度长**：真正的庄家会在币低位横盘期间（3天-2周）分批买入
2. **分仓操作**：大号→小号→多个中间号，避免被追踪
   - Transfer 进来 + Swap 卖出 = 有效的分仓信号（非空投）
   - Scanner v5已实现feePayer追踪+Union-Find实体合并
3. **持仓模式**：
   - 低位密集买入（价格CV<12%的横盘区间）
   - 单笔金额适中（$300-$3,000），不大不小
   - 买入间隔有人类特征（非固定秒级间隔 = 非bot）
4. **抛售模式**：
   - 在拉盘启动后阶梯卖出（不一次性dump）
   - 每次卖出比例<20%（保持价格不崩）
   - 卖出周期3-7天

### 检测庄家吸货 vs Scam的关键区分指标
| 指标 | 庄家吸货（跟） | Scam/Rug（避） |
|------|----------------|----------------|
| 持有时间 | >3天 | <1小时 |
| 买入模式 | 分批、不同金额 | 同一区块、等额 |
| 资金来源 | 多层中转但有真实历史 | 新钱包/burner |
| 卖出模式 | 阶梯慢卖 | 一次性dump |
| 是否有社交 | 可能有TG群/推特 | 无或虚假 |
| DEV行为 | 持续互动 | 消失 |

### Scanner v5 实际发现
- 45个钱包池，42个币扫描完毕
- 严格标准（≥3币+均利≥$2k+胜率≥60%）几乎无人达标
- **问题**：标准太高，把庄家和散户一起过滤了
- **优化方向**：
  1. 降低跨币要求：≥2币即可（专注某条链的庄家可能只做2-3个币）
  2. 加入"未实现利润"：持有但未卖出的盈利也算（大户可能还没跑）
  3. 按时间窗口分级：7天内活跃的权重更高
  4. 加入分仓关联：Union-Find合并后的实体PnL才是真实水平

### 关键洞察
1. **不要跟公开聪明钱列表**：GMGN/Birdeye上的列表alpha已被稀释，跟的人太多
2. **自建列表才有优势**：但建设周期长（scanner需要积累数据）
3. **最有价值的信号**：庄家从低位开始建仓 + 社交热度开始上升 = 即将拉盘
4. **98.6%的pump.fun币失败**，93%有操纵迹象 — 只做剩下的7%

## 9. 交易策略详细设计（2026-03-14 15:15）

### 资金管理
- **总资金**: $100（跑步哥定的起步金）
- **单笔**: $20（分5笔试错）
- **加仓规则**: 首笔盈利+50%以上且庄家未卖 → 追加$20（用利润加仓，不动本金）
- **极限亏损**: $50（5笔全亏-50%），触发后暂停24h复盘

### 入场条件（必须全部满足）
1. 评分≥60分（安全+链上+市场+情绪综合）
2. 币创建时间<4小时（越新越好，>4h打折，>24h不做）
3. 当前非深夜（UTC 06:00-22:00，对应北京14:00-06:00）— 美国时段流动性最好
4. SOL大盘24h不跌>5%（熊市不冲）
5. 手上持仓≤3个（避免分散注意力）

### 入场方式
- **买入**: 市价Swap via Jupiter/Raydium（速度优先）
- **滑点**: 设5%（meme波动大，太低成交不了）
- **Priority fee**: 0.001 SOL（保证速度但不烧太多）
- **确认**: 买入后5秒内查持仓确认到账

### 加仓规则
- **条件**: 首笔+50%且以下任一：
  - 庄家钱包还在加仓
  - 推特热度上升（搜索结果增加）
  - holder数1h内增>100
- **金额**: $20（用浮盈，不加本金）
- **最多加仓1次**（$100本金经不起多加）

### 止盈（阶梯式，写进评分模型第7节的扩展）
| 涨幅 | 动作 | 保留 |
|------|------|------|
| +100% (2x) | 卖40% | 60% |
| +400% (5x) | 卖30% | 30% |
| +900% (10x) | 卖20% | 10% |
| +1900% (20x) | 卖剩下的一半 | 5% |
| +4900% (50x+) | 剩5%放飞（moon bag） | — |

### 止损（硬性，不犹豫）
| 条件 | 动作 |
